# Lite Server Monitor (LSM)

> Легкий, модульный инструмент для мониторинга и базовой защиты серверов Ubuntu и Debian.

**Lite Server Monitor (LSM)** — это open-source проект, предназначенный для мониторинга состояния Linux-серверов без использования тяжелых систем мониторинга, баз данных, Docker и веб-интерфейсов.

Проект ориентирован на простоту установки, минимальное потребление ресурсов и надежную работу в автоматическом режиме.

После установки LSM самостоятельно контролирует состояние сервера, отправляет уведомления при возникновении проблем и формирует ежедневный отчет о состоянии системы.

---

# Возможности

## Мониторинг системы

- Мониторинг свободного места на дисках
- Мониторинг SMART
- Мониторинг RAID-массивов
- Мониторинг загрузки CPU
- Мониторинг использования оперативной памяти
- Мониторинг температуры процессора
- Мониторинг состояния UPS (APC / apcupsd)
- Ежедневный отчет о состоянии сервера

---

## Безопасность

- Мониторинг входов по SSH
- Статистика успешных входов
- Статистика неудачных попыток авторизации
- Интеграция с Fail2Ban
- Статистика заблокированных IP-адресов
- Security Summary в ежедневном отчете

---

## Уведомления

Поддерживаются следующие способы доставки уведомлений:

- Telegram
- Email (SMTP / msmtp)

Все уведомления проходят через единый модуль, что обеспечивает единый формат сообщений и упрощает сопровождение проекта.

---

# Почему Lite Server Monitor?

В отличие от крупных систем мониторинга, Lite Server Monitor не требует:

- Docker
- Kubernetes
- PostgreSQL
- MySQL
- Redis
- Web UI
- Агентов
- Сложной первоначальной настройки

После установки проект практически не требует обслуживания.

Основная идея проекта:

> **Установил → Настроил → Забыл.**

---

# Для каких серверов подходит

Lite Server Monitor отлично подходит для:

- домашних серверов;
- NAS;
- серверов печати;
- серверов видеонаблюдения;
- виртуальных машин;
- небольших офисных серверов;
- выделенных Linux-серверов.

---

# Чем этот проект НЕ является

Lite Server Monitor **не является заменой**:

- Zabbix
- Prometheus
- Grafana
- Netdata

Проект не предназначен для централизованного мониторинга сотен серверов.

Его задача — обеспечить простой, надежный и понятный мониторинг одного или нескольких Linux-серверов.

---

# Поддерживаемые операционные системы

В настоящее время поддерживаются:

- Ubuntu Server 22.04 LTS
- Ubuntu Server 24.04 LTS
- Debian 12

Список поддерживаемых систем будет расширяться.

# Архитектура проекта

Проект построен по модульному принципу.

Каждый модуль отвечает только за одну задачу и может быть включен или отключен независимо от остальных.

Основные компоненты проекта:

- CLI `lsm`
- модульный установщик
- библиотека общих функций
- независимые модули мониторинга
- единая система уведомлений
- systemd services и timers

---

- System Monitor

- Login Monitor

# Установка

После выхода первой стабильной версии установка будет выполняться одной командой.

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/<username>/Lite-Server-Monitor/main/bootstrap.sh)
```

Во время установки мастер предложит:

- выбрать необходимые модули;
- настроить Telegram;
- настроить Email;
- использовать готовые шаблоны конфигурации;
- либо выполнить настройку вручную.

# После установки

По умолчанию используются следующие каталоги.

| Каталог | Назначение |
|----------|------------|
| `/opt/lsm` | Файлы программы |
| `/etc/lsm` | Конфигурация |
| `/var/lib/lsm` | State-файлы |
| `/var/log/lsm` | Логи |
| `/run/lsm` | Lock-файлы |
| `/usr/local/bin/lsm` | CLI |

Подробная информация приведена в документе **INSTALLATION_LAYOUT_RU.md**.
