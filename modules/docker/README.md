# Модуль мониторинга Docker

## Описание

Модуль `docker` обеспечивает автоматический контроль работы демона Docker, отслеживает состояние запущенных и остановленных контейнеров, а также контролирует объем дискового пространства, занимаемого Docker (образами, контейнерами и томами).

## Возможности

- Проверка активности службы Docker (`docker.service`)
- Детекция неожиданно остановленных или упавших (exited) контейнеров
- Мониторинг дискового пространства, используемого подсетью Docker
- Гибкое отключение отдельных проверок через конфигурационный файл
- Уведомления через централизованную систему алертов LSM

## Требования

- Установленный и запущенный `docker`

## Конфигурация

```
/etc/lsm/modules/docker.conf
```

## Основные параметры:

- `DOCKER_ENABLED` — включение/выключение модуля (`true`/`false`)
- `CHECK_SERVICE` — проверка статуса службы Docker
- `CHECK_CONTAINERS` — проверка статусов контейнеров
- `CHECK_STORAGE` — проверка занимаемого диска
- `STOPPED_CONTAINER_WARNING` — предупреждать об остановленных контейнерах
- `STORAGE_WARNING_GB` — порог объёма диска в ГБ для отправки предупреждения

## Устанавливаемые файлы

```
/opt/lsm/modules/docker/check_docker.sh

/etc/systemd/system/lsm-docker.service

/etc/systemd/system/lsm-docker.timer

/etc/lsm/modules/docker.conf
```

## Расписание по умолчанию

Каждые 15 минут.
