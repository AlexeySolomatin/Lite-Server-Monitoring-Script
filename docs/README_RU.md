# Lite Server Monitoring Script

> Легкий, модульный и надежный набор Bash-скриптов для мониторинга серверов Ubuntu и Debian.

**Lite Server Monitoring Script** — это открытый проект, предназначенный для мониторинга состояния Linux-серверов без использования Docker, баз данных, веб-интерфейсов и сложных систем мониторинга.

Основная идея проекта — предоставить системному администратору простой инструмент, который можно установить за несколько минут и получать только действительно важные уведомления о состоянии сервера.

---

# Почему появился этот проект?

Существует множество систем мониторинга, однако большинство из них ориентированы на крупную инфраструктуру и требуют значительных ресурсов.

Наиболее распространенные недостатки таких решений:

- сложная установка;
- необходимость отдельной базы данных;
- использование Docker или Kubernetes;
- веб-интерфейс, который нужен далеко не всегда;
- большое количество служб и фоновых процессов;
- высокая сложность сопровождения.

Для домашних серверов, небольших организаций и специализированных серверов подобные решения зачастую избыточны.

Lite Server Monitoring Script создается как простая альтернатива, которая выполняет только одну задачу — своевременно сообщает администратору о возникшей проблеме.

---

# Для каких серверов подходит проект?

Проект отлично подойдет для мониторинга:

- домашних серверов;
- NAS;
- серверов печати;
- серверов видеонаблюдения;
- виртуальных машин;
- небольших офисных серверов;
- выделенных Linux-серверов;
- мини-ПК на Ubuntu Server или Debian.

---

# Чем этот проект НЕ является

Lite Server Monitoring Script — это не замена Zabbix, Prometheus, Grafana или Netdata.

Проект не предназначен для построения централизованной системы мониторинга сотен серверов.

Его задача — предоставить простой, надежный и легкий инструмент для мониторинга одного или нескольких Linux-серверов с минимальными требованиями к системе.

# Возможности

## Мониторинг накопителей

- контроль свободного места;
- контроль состояния SMART;
- контроль RAID-массивов;
- уведомление о восстановлении после ошибки;
- защита от повторной отправки одинаковых уведомлений.

---

## Мониторинг системы

- загрузка процессора;
- использование оперативной памяти;
- системная нагрузка (Load Average);
- температура процессора.

---

## Мониторинг ИБП

- поддержка APC UPS через apcupsd;
- уведомление об отключении питания;
- уведомление о переходе на батарею;
- уведомление о восстановлении электропитания;
- уведомление перед безопасным выключением.

---

## Уведомления

Поддерживаются:

- Telegram;
- Email (SMTP);
- ежедневный отчет о состоянии сервера.

---

## Надежность

Все модули проекта используют единые принципы работы:

- защита от параллельного запуска (flock);
- state-файлы;
- Recovery Notification;
- защита от спама уведомлениями;
- единая система отправки сообщений.

---

# Основные принципы проекта

Проект создается с учетом следующих требований:

- простота установки;
- минимальное количество зависимостей;
- отсутствие базы данных;
- отсутствие Docker;

# Установка

После выхода первой стабильной версии установка будет выполняться одной командой:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/<username>/Lite-Server-Monitoring-Script/main/bootstrap.sh)
```

Установщик автоматически:

- проверит операционную систему;
- установит необходимые зависимости;
- предложит выбрать необходимые модули;
- настроит уведомления;
- выполнит проверку работоспособности;
- подготовит систему к эксплуатации.
